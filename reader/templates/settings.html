Speech Text Processing
Converts digits, dates, and currency into spoken form (e.g. “2345” → “two thousand three hundred forty-five”). English/Chinese/Japanese: install wetext on Windows (or WeTextProcessing on Linux/macOS). Other languages (including Hungarian) use num2words for integers. Changing this creates new audio cache entries for affected segments.
OmniVoice Generation & Export
Used for playback and export. Lower steps are much faster; 16 is usually enough. Changing this creates new audio cache entries (existing files are kept).
How many coalesced utterances go into one OmniVoice.generate() call. OmniVoice is only ~0.6B params — even batch 24 often uses just a few GB and Task Manager “3D” load stays low. Check the terminal for GPU generate done: batch=… peak_alloc=… (that is the truth). Prefer nvidia-smi / Task Manager “CUDA” engine, not “3D”.
Joins consecutive same-voice sentences into one longer synth, then splits the audio back for subtitles/timeline. This usually speeds full-book export more than raising batch size alone (paper: batch 1→8 only ~30% faster).
On the tested RTX 3090 + Windows 11 setup, plain PyTorch was slightly faster, so use Off unless your own benchmark proves otherwise. Full Triton/hybrid needs omnivoice-triton (officially WSL2/Linux; on native Windows try pip install triton-windows then pip install omnivoice-triton).
